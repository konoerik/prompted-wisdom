## Project Overview

A beautifully designed, multi-page web resource that distills humanity's greatest wisdom for young people. The content is pre-generated from multiple LLMs using the same prompt, allowing readers to toggle between "versions" of the guide тАФ like switching Python documentation versions тАФ and observe how different AI minds interpret the same human questions.

---

## The Core Prompt (used across all LLMs)

> You are a journalist for a respectable magazine. You have maintained an advice column for 20 years and your readers love your work. You are not only a student of philosophical ideas, psychology and history, but your travels have exposed you to how these ideas materialize across the world. You are now distilling everything into a concise guide on how to live your life тАФ written directly to a young person of 20. You write clearly, without jargon, and you do not moralize. You share what you have observed to be true. Your goal is to give them an anchor for the years ahead.

Each chapter is generated by appending a chapter instruction to the core prompt, for example:

> Now write the chapter: On Knowing Yourself. Aim for approximately 700 words.

### Word Count Guidelines by Chapter

| Chapter | Target Length |
|---|---|
| What the Greatest Thinkers Taught Us | 900тАУ1000 words |
| All middle chapters (2тАУ9) | 600тАУ800 words |
| A Letter to You | 400тАУ500 words |

The shorter closing letter is intentional тАФ closing letters lose power when they run long.

## Methodology & Experimental Controls

To ensure meaningful comparisons between models and between runs:

- **Temperature = 0** тАФ makes each model deterministic. Same prompt, same output every time. Eliminates run-to-run variation within a model.
- **Fixed max tokens** тАФ identical ceiling for every model. No model wins by writing more.
- **Identical chapter prompts** тАФ the follow-up instruction is word-for-word identical across every model.
- **No additional system prompts** тАФ extra instructions interact differently with each model's training. One clean input only.
- **Fixed output structure** тАФ models are instructed to use consistent section formatting so outputs are structurally comparable.

### Provenance & Integrity

There is currently no tamper-proof way to prove an LLM generated a specific piece of text. The approach taken here is transparency and audit trail:

- Full API response metadata stored in each JSON file (model version, timestamp, token counts)
- SHA-256 hash of the raw output generated and stored at time of creation
- Raw API parameters documented in METHODOLOGY.md
- All content version controlled via git тАФ full history publicly visible

Each JSON file includes:

```json
{
  "model": "claude",
  "model_version": "claude-sonnet-4-20250514",
  "generated_at": "2026-04-10T14:32:00Z",
  "input_tokens": 312,
  "output_tokens": 687,
  "content_hash": "sha256:a3f9...",
  "body": [...]
}
```

## UI / UX Concept

### Model Toggle (Top of Page)
- A dropdown or tab switcher at the top of every page
- Options driven dynamically by `/meta/models.json` тАФ adding a new model requires no code changes
- Selecting a model swaps the page content via JavaScript тАФ no page reload
- Inspired by Python documentation version switcher
- All content is pre-generated and stored as JSON тАФ no live API calls

### Navigation
- Left sidebar (desktop) or hamburger menu (mobile) with all chapter titles
- Active chapter highlighted
- Clean, readable, editorial aesthetic

### Content Pages
Each page follows the same structure:
1. Title
2. Main body text (the LLM-generated wisdom)
3. **Resources section** at the bottom тАФ curated links to free, distraction-free sources

## Aesthetic Direction

**Editorial / Magazine meets Ancient Library**

- Think: a serious literary journal crossed with a well-worn philosophy textbook
- Dark, warm background (deep ink, parchment, or dark olive tones) тАФ NOT a purple gradient
- Serif display font for headings (Cormorant Garant, Playfair Display, or similar тАФ load from Google Fonts)
- Clean readable body font (Georgia or a refined humanist serif)
- Subtle paper/grain texture overlay
- Gold or amber accent color
- Generous whitespace, long-form reading optimized
- Chapter numbers rendered large and faint in the background
- Tasteful animations: fade-in on page load, smooth transitions between model toggles
- Max line width ~70 characters for comfortable long-form reading

## Resources Section (Per Page)

### Editorial Philosophy
Every linked resource must be:
- **Free to access** тАФ no paywalls, no subscriptions
- **A destination, not a gateway** тАФ no algorithmic feeds, no autoplay, no recommendations engines, no comments sections
- **Distraction-free** тАФ the reader should be able to go there, read, and return

This rules out YouTube and social platforms by design. The goal is to give young people an anchor, not another current to get swept into.

### Approved Sources
- **Project Gutenberg** (gutenberg.org) тАФ primary texts, public domain classics
- **Standard Ebooks** (standardebooks.org) тАФ same texts, beautifully formatted and corrected
- **Internet Archive** (archive.org) тАФ broader coverage, harder to find works
- **Wikisource** (wikisource.org) тАФ essays, speeches, shorter works
- **LibriVox** (librivox.org) тАФ free audiobooks of public domain texts
- **Stanford Encyclopedia of Philosophy** (plato.stanford.edu) тАФ rigorous, free, written by philosophers. No rabbit holes.

### Resource Block Structure (per chapter)
```
ЁЯУЪ Read
ЁЯОз Listen
ЁЯУД Essays & References
```

## Technical Spec

### Stack
- **Pure HTML5, CSS3, vanilla JavaScript** тАФ no frameworks, no build step, no Node
- **Deployment**: GitHub Pages тАФ push and done
- **Content**: JSON files, one per chapter per model, loaded via `fetch()`
- **Routing**: Single `index.html` тАФ JavaScript handles chapter and model display based on URL hash (e.g. `index.html#claude/knowing-yourself`)

### How the Model Toggle Works
```javascript
// When user selects a model or chapter:
fetch(`/content/${selectedModel}/${currentChapter}.json`)
  .then(r => r.json())
  .then(data => renderPage(data));
```
No server needed. GitHub Pages serves the JSON files as static assets.

### models.json тАФ Model Registry
Drives the toggle dropdown dynamically. Adding a new model requires no code changes:

```json
[
  {
    "id": "claude",
    "label": "Claude",
    "version": "claude-sonnet-4-20250514",
    "generated_at": "2026-04-10",
    "status": "complete"
  },
  {
    "id": "gpt4",
    "label": "GPT-4",
    "version": "gpt-4o",
    "generated_at": "",
    "status": "pending"
  }
]
```

## Community Contributions

This project welcomes contributions of model outputs. To contribute:

1. Use the canonical prompt in `PROMPT.md` тАФ word for word, no modifications
2. Use Temperature = 0 and document your API parameters in your PR
3. Store the full API response metadata alongside your content
4. Generate a SHA-256 hash of the raw output and include it
5. Follow the JSON structure exactly as specified above
6. Submit via GitHub pull request тАФ one chapter per PR for easy review

The `models.json` registry and folder structure are designed so that adding a new model requires no changes to the codebase тАФ only new content files and one registry entry.
